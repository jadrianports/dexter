<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YouTube metadata → `decide_crossfade` | `Track.duration_seconds` is attacker-influenceable (a video uploader controls its own metadata) and is known to disagree with the real file (RESEARCH landmine #5). It reaches this seam as an int. |
| `config.py` → seam defaults | Trusted, in-process, not user-settable (D-12b: no `/crossfade [seconds]` arg exists, so no untrusted int can reach `fade_seconds`). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-01 | Tampering | `cut_frame` fed hostile/wrong `outgoing_duration` | mitigate | Floor the result at 0 so a duration shorter than the fade can never produce a negative frame index (which downstream becomes an `-ss` past EOF). Locked by `test_cut_frame`. |
| T-27-02 | Denial of Service | `decide_crossfade` | accept | Pure arithmetic over primitives, no loops, no allocation, no I/O — constant time. No exhaustion surface. |
| T-27-03 | Elevation of Privilege | `logic/crossfade.py` import surface | mitigate | Purity contract enforced by an AST scan in Task 2's verify: zero `discord` imports. The seam cannot reach a voice client, a socket, or the DB even if a later edit tried. |
| T-27-04 | Information Disclosure | `FadeVerdict` value strings reaching logs | accept | Values are fixed internal literals (`"not_cached"` etc.) chosen at authoring time — no user data, no track titles, no IDs are interpolated into them. |
| T-27-SC | Tampering | npm/pip/cargo installs | mitigate | **Zero installs this plan.** `audioop` is stdlib on the pinned 3.11 (D-02); no `requirements.txt` change. No package-legitimacy checkpoint is required because no package is added. |
