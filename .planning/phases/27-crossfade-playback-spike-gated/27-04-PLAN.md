<objective>
Ship D-17's **mandatory guard rails**: a wrapped, fail-soft install of the `AudioPlayer.send_silence`
suppression patch, and a CI drift guard that asserts **the call site**, not merely the method's
existence. Plus the one-line latent LFU fix that stops a fade's outgoing file from being
eviction-eligible.

Purpose: D-17 approved the suppression **only** with both rails. Un-suppressed, the 100ms boundary
lands mid-music at −9.8 dBFS and crossfade sounds *worse* than the hard cut it replaces — but the
patch touches an undocumented library internal (`discord/player.py`), and a bare
`_original = AudioPlayer.send_silence` raises `AttributeError` **at import**, which is a boot crash.
Both rails convert that failure mode into "the 100ms comes back" plus a red build.

**This is genuinely novel structure.** The pattern mapper grepped `bot.py`, `cogs/`, `services/`,
`utils/`, `logic/` for `hasattr(discord`, `getattr(discord`, `monkeypatch`, `_original_` and found
**zero hits** — this codebase has never patched or capability-checked a third-party internal in
production code. There is no house style to copy. Plan and review it accordingly.
Output: one isolated patch module, one boot-time install site, one drift guard with a positive
control, one latent-bug fix.

Covers VALIDATION.md Behavior Map rows **17, 18** + RESEARCH landmine #3 / Open Question 8.
</objective>
