<objective>
Run the planner-owned numeric-defaults validation spike that OPENS the retrieval work. The retrieval
defaults (top-k, similarity floor, dedup threshold, per-user cap, decay window, rerank weights, inject
cap, callback chance) are MEDIUM-confidence priors (Assumption A3). Convert them into chosen, recorded
constants BEFORE 11-03 retrieval lands, using this bot's actual `gemini-embedding-001` @ 768d behavior
on a small representative corpus.

Purpose: "no memory beats a wrong memory" (Pitfall 8) — the similarity floor and rerank weights decide
recall quality, and they are the one genuine open variable in an otherwise HIGH-confidence phase
(CONTEXT: Claude's Discretion; RESEARCH § Validation Architecture). This is the empirical seam.

Output: `scripts/memory_spike.py` (throwaway) and config.py constants updated to the empirically
chosen values, each annotated `# tuned via 11-02 spike <date>`. The pure rerank functions built in
11-03 are parameterized by these constants, so changing a value never breaks a unit test.
</objective>

    (1) defines a small hand-written seed corpus of ~12-20 distilled, episode-style facts (NO numbers,
    third-person, e.g. "swore he was done with the killers", "only queues sad songs after midnight")
    spread across 2 fake user_ids, plus a parallel set of ~6 irrelevant decoy facts; (2) embeds them
    with `client.aio.models.embed_content(model=config.EMBEDDING_MODEL,
    config=types.EmbedContentConfig(output_dimensionality=config.EMBED_DIM, task_type="RETRIEVAL_DOCUMENT"))`
    and inserts them into `user_memories` via a pool created with `init=register_vector` (reuse the
    11-01 boot ordering — extension already exists), or a disposable temp table; (3) embeds ~8 realistic
    roast-query strings with `task_type="RETRIEVAL_QUERY"`; (4) runs the scoped cosine search
    (`ORDER BY embedding <=> $query LIMIT MEMORY_TOP_K`, `1 - (embedding <=> $query) AS similarity`,
    scoped `WHERE user_id = $1`); (5) prints, per query: the ranked (fact, similarity) list, the
    similarity gap between the most-relevant decoy and the least-relevant true match (the floor
    signal), and pairwise similarity of near-duplicate seed facts (the dedup-threshold signal). Read
    the API key from `os.getenv("GEMINI_API_KEY")` and the DSN via `config.sanitize_database_url`.
    Clean up any temp rows it inserts (delete by the fake user_ids) so the spike is non-destructive.
