<objective>
Build the producer half of the write path and its safety gates: a constrained distillation step that
turns raw banter / notable-event context into 0-3 atomic, third-person, NUMBER-FREE episode facts; a
STOP-SHIP sensitivity/PII gate that drops identity & wellbeing content (D-01..D-03); and the two — and
only two — write triggers (notable-event hooks + once-daily batch, D-09). Wire `distill_and_remember()`
into the already-firing notable-event sites and add the daily batch task. No per-message writes (D-09),
no voice-session-end trigger (D-10).

Purpose: MEM-04 (event/batch triggers, never per-message) + MEM-05 (sensitivity/PII stop-ship gate +
accuracy firewall: never embed numbers SQL already knows). This is the headline HIGH-severity security
work for the phase — durable storage of personal data is a retention liability.

Output: `distill()` + `distill_and_remember()`, the `is_sensitive`/`contains_number` pure gates, the
`memory_distill_batch` daily loop, and fire-and-forget hooks at the repeat-song / milestone / streak /
late-night / auto-queue-ignored surfaces.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user banter → distiller (LLM) | untrusted chat text becomes a durable stored fact |
| distiller output → store | LLM output crosses into permanent retention |
| cog event handler → create_task | the 3s interaction / non-blocking boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-05a | Information Disclosure (HIGH) | durable storage of sensitive identity/wellbeing/PII content | mitigate | STOP-SHIP gate: DISTILL_PROMPT forbids the D-01 categories AND a deterministic is_sensitive() backstop drops any surviving blocked fact before remember(); ships in v1 (D-03), never deferred |
| T-11-05b | Tampering (accuracy) | embedding a number SQL already answers (stale-count drift) | mitigate | DISTILL_PROMPT forbids numbers + contains_number() backstop drops number-bearing facts (Critical Rule 5 / Pitfall 5) |
| T-11-05c | Spoofing (fact provenance) | hallucinated "facts" stored as truth | mitigate | distiller constrained to observed/stated, atomic, third-person, no inference (V5); candidate-ammo framing lets the model NOOP downstream |
| T-11-05d | Denial of Service | per-message distillation floods quota / context rot | mitigate | only two triggers (event hooks + daily batch), one batched priority-2 call; no per-message write (D-09); on_message regression test enforces it |
| T-11-05e | Denial of Service | inline await blocks an interaction/event handler | mitigate | every cog write is asyncio.create_task fire-and-forget (3s rule) |
</threat_model>
