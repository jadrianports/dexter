<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Any Discord user (guild or DM) ‚Üí `/invite` | Untrusted invocation crosses here. The command takes NO user input (no parameters), so there is no injection surface ‚Äî but it does emit a URL that others will click. |
| `/invite`'s emitted URL ‚Üí the invitee's OAuth2 consent screen | Whatever this URL says is what the invitee grants. A drifted or hand-built URL here is a real privilege-grant bug, not a cosmetic one. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | Elevation of Privilege | `cogs/invite.py`'s emitted URL | mitigate | The cog never constructs a URL ‚Äî it calls `logic.invite.build_invite_url()` with `config.INVITE_PERMISSIONS_VALUE` (the D-02-locked bitfield from 22-01). `test_cog_does_not_construct_a_url_itself` asserts no `oauth2/authorize` literal and no `oauth_url(` call exists in the cog, so a future edit cannot quietly widen the grant here. |
| T-22-02 | Tampering | The in-bot copy of the promoted link | mitigate | `test_invite_command_sends_the_canonical_url` asserts the button's `url` is byte-equal to `build_invite_url()`'s output, making the in-bot copy the single source of truth SC-3 requires. Plan 22-03 extends the same guarantee to git-tracked docs. |
| T-22-05 | Spoofing | The link button's destination | mitigate | D-07: the button URL is the literal `https://discord.com/oauth2/authorize?...` endpoint ‚Äî no shortener, no vanity redirect, no third-party hop that could be repointed. Asserted by 22-01's `test_url_is_the_literal_oauth2_endpoint` and re-asserted at the button level here. |
| T-22-06 | Denial of Service | `/invite` has no cooldown | accept | Deliberate (Claude's Discretion, CONTEXT.md). The handler does zero I/O ‚Äî no DB, no Gemini, no network ‚Äî and returns a static string built in microseconds. Discord's own per-application interaction rate limits are the backstop. Adding a cooldown would buy nothing and would contradict D-05's "spreading the bot is the point". |
| T-22-07 | Information Disclosure | `/invite` usable in DMs | accept | The URL is public promotional material by design; there is nothing to disclose. DM invocation flows through the existing `DexterCommandTree.interaction_check` choke point unchanged (`has_guild=False` is a first-class allowed case in `decide_interaction_allowed`) ‚Äî no new auth path is introduced (ASVS V2/V4: no change). |
| T-22-SC | Tampering | Supply chain (pip installs) | accept | **Zero new package installs this phase.** No `pip install` task exists in this plan. |
