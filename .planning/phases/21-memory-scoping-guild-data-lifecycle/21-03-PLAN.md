<objective>
Wire the opt-ins. Five recall call sites opt into guild-scoping (`guild_scoped=True`); exactly one
— `/ask` — stays global and is deliberately NOT edited (MEM-02).

Purpose: this is where MEM-01 actually ships. The substrate from 21-01 is inert until these call
sites ask for it. It is also where a lazy shortcut would silently break MEM-02, so the discipline
below is non-negotiable.

The discriminator (resolved by research, do not relitigate): **"explicit synchronous self-pull"
stays global; "unprompted ambient broadcast to a channel" scopes.** It is NOT "self vs. third
party by user_id" — under that (wrong) rule the two `events.py` surfaces and the `music.py`
callback all recall the SAME user the event is about and would be misclassified as safe.

Output: 5 opt-ins, 1 deliberate non-edit, and per-call-site tests locking both.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Guild A memory → Guild B channel | The actual leak MEM-01 names. Crossed whenever an unprompted surface broadcasts a recalled memory into a channel of a guild other than the one the memory was formed in. |
| `/ask` invoker → their own memory | NOT a trust boundary — a user pulling their own memory cannot expose anyone else. Scoping it would be a functionality loss with zero security gain (MEM-02). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-04 | Information Disclosure | The 6 `recall()` call sites | mitigate | Each of the 5 ambient/broadcast sites carries an explicit `guild_scoped=True` literal; `/ask` carries no kwarg at all. Task 3 locks each site individually (behavioral kwarg assertions for `/roast`, `/ask`, ambient roast; source assertions for proactive, music, auto-queue). A copy-paste that scoped `/ask` or un-scoped `/roast` fails a named test. |
| T-21-08 | Information Disclosure | Music-callback empty-string guild_id | mitigate | `_build_roast_line`'s guild_id param defaults to `None`. Passing `guild_scoped=True` with an empty-string guild_id would emit `(guild_id = '' OR guild_id IS NULL)` — silently narrowing recall to the legacy NULL corpus and quietly degrading the feature. `guild_scoped=bool(guild_id)` makes the opt-in conditional on actually having a guild. |
| T-21-09 | Information Disclosure | Auto-queue cross-guild taste bleed | mitigate | Today a member's Guild-A taste episodes are blended into Guild B's room recommendation the moment they join Guild B's voice channel — cross-guild travel mediated through a recommendation rather than a visible roast. `guild_scoped=True` on the taste-blend recall closes it. `taste_episode` facts are already guild-stamped at write time (Phase 13 D-06/D-07), so there is no NULL-corpus concern for this kind. |
| T-21-SC | Tampering | npm/pip/cargo installs | accept | Zero packages installed this phase. No supply-chain surface. |
</threat_model>
