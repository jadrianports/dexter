<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Discord gateway (guild removal) → DB purge | A guild departure triggers destructive, irreversible DELETEs. The blast radius must be exactly the departing guild's data. |
| Purge → abuse-mitigation control (`guild_blocklist`) | The blocklist is the ONLY thing preventing a kicked abuser from re-inviting. It must survive the purge that the abuser's own removal triggers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-03 | Tampering / Denial of an abuse-mitigation control | `database.purge_guild_data` table list | mitigate | The table list is four hardcoded SQL literals — never a loop, never `information_schema` introspection, never a parameterized table name. Task 1's verify command greps the function source for `guild_blocklist` (must be absent) and counts `DELETE FROM` (must be exactly 4). Task 2's live-DB test inserts a `guild_blocklist` row for the target guild BEFORE the purge and asserts it survives. Regression here = a blocked guild can re-invite (OWNER-04 broken). |
| T-21-06 | Information Disclosure (data resurfacing) | Purge completeness | mitigate | All four tables are purged in ONE transaction — a partial purge (e.g. `guild_jams` deleted but `user_memories` retained) would leave stale context that resurfaces on re-invite, which is precisely the MEM-04 failure mode. Atomicity means all-or-nothing. |
| T-21-07 | Denial of Service (destructive over-deletion) | `user_memories` NULL corpus | mitigate | `WHERE guild_id = $1` uses SQL three-valued logic: `=` never matches NULL, so the D-01 grandfathered global corpus is excluded with no extra clause. Task 2 asserts a `guild_id IS NULL` memory survives the purge. An over-broad purge (e.g. `WHERE guild_id = $1 OR guild_id IS NULL`) would destroy every user's `daily_batch` memories globally on ANY guild departure. |
| T-21-SC | Tampering | npm/pip/cargo installs | accept | Zero packages installed this phase. No supply-chain surface. |
</threat_model>
