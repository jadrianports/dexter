<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| song title/artist -> outbound HTTP | User-influenced track title/artist are used to build the AZLyrics URL and the Genius search query. |
| scraped HTML -> Discord embed | Lyrics HTML from Genius/AZLyrics is untrusted content that will be rendered in a Discord embed (by plan 03-05). |
| GENIUS_TOKEN secret | Outbound auth token read from env. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering (path traversal / SSRF) | build_azlyrics_url | mitigate | `re.sub(r"[^a-z0-9]", "", x.lower())` strips ALL non-alphanum from artist+song; URL host is the hard-coded azlyrics.com — no user-supplied host reaches the HTTP client (no SSRF) |
| T-03-07 | Spoofing / Injection | scraped lyrics -> embed | mitigate | sanitize_lyrics runs BeautifulSoup get_text (strips `<script>`/tags) and neutralizes @everyone/@here before the text is returned; plan 03-05 additionally sets allowed_mentions=none on send |
| T-03-08 | DoS | _get_azlyrics fetch | mitigate | aiohttp.ClientTimeout(total=10) caps fetch time; `len(html) > 500_000` cap prevents memory exhaustion; Genius retries=1 + timeout=5 limit amplification |
| T-03-09 | Information Disclosure | GENIUS_TOKEN | mitigate | Token read from env, passed only to Genius() constructor, stored as a private attr, never logged or echoed in any error message/embed |
| T-03-10 | Tampering (supply chain) | new PyPI installs | mitigate | Blocking-human legitimacy checkpoint (T-03-SC) verifies all four [ASSUMED] packages on PyPI before install |
| T-03-SC | Tampering | pip installs (lyricsgenius/beautifulsoup4/aiohttp/tzdata) | mitigate | slopcheck unavailable at research time -> all [ASSUMED]; blocking-human checkpoint verifies each on pypi.org before requirements.txt is touched |
