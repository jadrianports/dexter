<objective>
Build `services/guild_config.py::GuildConfigService` (D-04) — the cache-owning I/O tier that holds
both named resolvers (D-02). It loads all config rows once at boot (D-06), serves them from an
in-memory cache with no per-event round-trip (CONFIG-03), fails closed on load error (D-07), and
seeds the home guild (CONFIG-05). `resolve_ambient_channel` is the STRICT row-only resolver (D-01)
that returns `None` (silent + WARNING log, row intact) on any uncertainty (D-03); it is SYNCHRONOUS
because it is cache-only. `resolve_announce_channel` is the SECOND, separately named resolver — the
preserved 4-step fallback chain (D-02) with ZERO callers this phase (a pre-built seam for Phase 19).

Purpose: One service owns the cache, the DB helper call sites, and both resolvers, giving Phases
19-21 one obvious extension point. The strict resolver makes multi-tenant silence structural.
Output: `services/guild_config.py`, `tests/test_guild_config_service.py`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| in-memory cache → ambient dispatch | The cache is the hot-path read tier; a wrong lookup could fire ambient output in the wrong guild |
| service → Postgres | seed writes cross into SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-XGUILD | Information Disclosure | `GuildConfigService._cache` / `get` | mitigate | Cache is keyed by `str(guild_id)`; `get(guild_id)` returns only that guild's row — no code path returns another guild's config; resolver only ever consults `guild.get_channel` for the SAME guild |
| T-18-STALE | Denial of Service (soft) | `resolve_ambient_channel` stale/unwritable branch | accept | Intentional per D-03 — a deleted channel or lost `send_messages` yields silence + a `log.warning`, and the row stays `configured=true` so recreating the channel / re-granting permission just works; "boring Dexter over broken Dexter" |
| T-18-FAILCLOSED | Denial of Service (soft) | `load_all` fail-closed branch | accept | Intentional per D-07 — a boot-load failure leaves every guild unconfigured (ambient silent, core commands live) rather than fail-open to the pre-refactor single-channel world; error surfaces to `dexter.log` + `ERROR_LOG_CHANNEL_ID` |
| T-18-SEED | Tampering | `seed_home_guild` | mitigate | Delegates to `seed_guild_config_if_absent` (`$N` params, `DO NOTHING`); refreshes only the seeded guild's single cache entry |
</threat_model>
