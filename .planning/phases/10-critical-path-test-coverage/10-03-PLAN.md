<objective>
Extract the EventsCog ambient-roast trigger/gating decision into pure, importable functions
in `logic/roasts.py` and lock it with pure-unit tests (TEST-03).

The nested trigger logic in `on_voice_state_update` (chance roll -> per-user cooldown ->
late-night eligibility -> second late-night roll -> join/leave scenario selection) becomes a
deterministic `decide_ambient_roast(...)` that takes the two `random.random()` rolls, the
`ZoneInfo`-computed `local_hour`, and the cooldown delta as IN parameters (D-06). The
`_check_ambient_cooldown` clock read becomes `cooldown_elapsed(seconds_since_last,
ceiling)`. The Gemini generation, channel resolution, `channel.send`, `_mark_ambient_roast`,
and the bot-move early-return stay untested-by-design glue. The live cog then CALLS the pure
functions as single source of truth (D-02).

Purpose: lock the roast gating so chance/cooldown/late-night boundaries can be exhaustively
tested with no `random`/clock patching. Adds no features, changes no behavior.
Output: `logic/roasts.py`, behavior-preserving edits to `cogs/events.py`, and
`tests/test_roast_logic.py`.

Note on parallelism: `logic/__init__.py` (package marker) is created by 10-01. This plan
creates only `logic/roasts.py` and does NOT touch `__init__.py`; `logic/` imports as a
namespace package regardless of marker ordering, so 10-01/10-02/10-03 run in parallel
(Wave 1) with zero file overlap and no hard dependency.
</objective>
