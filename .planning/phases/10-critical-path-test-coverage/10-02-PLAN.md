<objective>
Extract the OpsCog metrics + `/health` status-determination decision logic into pure,
importable functions in `logic/health.py` and lock it with pure-unit tests (TEST-02).

Two decisions are carved out of the async glue: the `degraded_reasons` -> HTTP-status
mapping currently inline in `bot.py`'s `/health` handler (becomes
`determine_health_status`), and the reason-assembly subset of `gather_bot_metrics` (becomes
`assemble_degraded_reasons`). The async DB probe, `bot.is_ready()` checks, and aiohttp
response plumbing stay untested-by-design glue. The live handler + metrics gatherer then
CALL the pure functions as the single source of truth (D-02). This covers the REL-01
degraded path Phase 9 added — scar #3 (D-05).

Purpose: lock the truthful-`/health` decision so it can never silently regress to "ok while
broken". Adds no features, changes no behavior.
Output: `logic/health.py`, behavior-preserving edits to `bot.py` + `cogs/ops.py`, and
`tests/test_health_logic.py`.

Note on parallelism: `logic/__init__.py` (package marker) is created by 10-01. This plan
creates only `logic/health.py` and does NOT touch `__init__.py`; `logic/` imports as a
namespace package regardless of marker ordering, so 10-01/10-02/10-03 run in parallel
(Wave 1) with zero file overlap and no hard dependency.
</objective>
