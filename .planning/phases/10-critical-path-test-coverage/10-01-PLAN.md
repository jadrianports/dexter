<objective>
Extract MusicCog playback decision logic into pure, importable functions in a new
`logic/playback.py` module and lock it with exhaustive pure-unit tests (TEST-01).

This carves the already-shipped decision cores — the `_on_track_end`
advance/loop/auto-queue-vs-stop branch, the auto-queue playback-start gate, and the
`restore_queues` index clamp / smart-rejoin / cap-truncate decisions — out of their async
Discord/DB glue so they can be tested deterministically, then makes the live cogs CALL
those pure functions as the single source of truth (D-02 true extraction — no mirrored
logic). Adds no features, changes no behavior. The exact pure/glue cut-line and the
function signatures below are the planner's derivation per D-07 (keyword-only primitives
chosen over a snapshot dataclass — each function has a small, cohesive arg list).

Purpose: lock decision logic that has already shipped live bugs. Three of the four phase
scar regression tests live here (D-05): finished-song replay (#1), silent auto-queue (#2),
restore index clamp (#4).
Output: `logic/__init__.py`, `logic/playback.py`, behavior-preserving edits to
`cogs/music.py` + `cogs/ai.py` + `services/queue_persistence.py`, and
`tests/test_playback_logic.py`.

Note on parallelism: `logic/__init__.py` (the package marker) is created here. Plans 10-02
and 10-03 create sibling `logic/` modules but do NOT touch `__init__.py`; Python treats
`logic/` as an importable namespace package even before the marker lands, so all three
Wave-1 plans run in parallel with zero file overlap and no hard ordering.
</objective>
