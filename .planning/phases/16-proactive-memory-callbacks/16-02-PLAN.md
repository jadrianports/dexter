<objective>
Build the opt-out storage substrate for PROACT-02: one additive boolean column on
`user_profiles` plus a getter/setter pair in `database.py`, mirroring the Phase 8 `total_errors`
additive-column precedent and the `update_user_profile` upsert shape. This is the store the
`/memory callbacks` command writes (plan 16-04) and the `on_message` gate reads (plan 16-03).

Purpose: The opt-out is a trust control (PROACT-02) that must be honored instantly and
completely, and must be structurally distinct from `/memory forget` â€” it flips a flag and
touches ZERO memory rows. The setter must be an upsert (not a bare UPDATE) because a user can
run `/memory callbacks` before ever queuing a song, when no `user_profiles` row exists yet
(Pitfall 3).
Output: `user_profiles.proactive_opt_out` column, `get_proactive_opt_out`,
`set_proactive_opt_out`, `tests/test_database_phase16.py`.
</objective>
