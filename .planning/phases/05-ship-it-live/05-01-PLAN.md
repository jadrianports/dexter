<objective>
Wire Dexter's database layer and HTTP liveness for the Koyeb + Neon substrate. This is the only material net-new application code in the phase: a pure `DATABASE_URL` sanitizer with Wave-0 unit tests (K-05), Neon-tuned asyncpg pool parameters at the `create_pool` call site (K-04), the cache-cap drop (K-07), and a minimal aiohttp `/health` endpoint that lets the bot deploy as a Koyeb WEB service and stay awake under an UptimeRobot ping (K-02 amended).

Purpose: Neon's pooled (PgBouncer transaction-mode) endpoint with 5-minute scale-to-zero breaks default asyncpg in three documented ways (unparseable `channel_binding`, SSL-EOF on reused dead connections, prepared-statement errors). Koyeb free offers no Worker services, so a long-lived bot needs an HTTP port. Without this plan the bot cannot connect to Neon or deploy on Koyeb at all.

Output: `config.sanitize_database_url`, constants `DB_MAX_INACTIVE_CONN_LIFETIME`/`DB_STATEMENT_CACHE_SIZE` plus updated `DB_POOL_MAX`/`AUDIO_CACHE_MAX_MB`, `tests/test_config.py`, the tuned `bot.py` `create_pool` call, and `bot.py._run_health_server()` launched as a background task.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Koyeb env → bot process | DATABASE_URL (Neon connection string, incl. credentials) crosses from Koyeb encrypted secret into the process at startup |
| Internet → /health endpoint | Koyeb health checker + UptimeRobot + any anonymous internet client can hit 0.0.0.0:8000/health |
| bot process → Neon | All DB traffic crosses the public internet to Neon's pooled endpoint |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | /health endpoint | mitigate | `/health` returns only the literal `{"status":"ok"}` — no DB host/credentials, no version, no bot/guild state, no DB ping. Endpoint is read-only GET, no params consumed. |
| T-05-02 | Information Disclosure | DATABASE_URL in logs | mitigate | create_pool call site keeps the existing `# SECURITY (T-04-05)` rule: never log the DSN or pool object. sanitize_database_url output is not logged. |
| T-05-03 | Tampering | SSL to Neon | mitigate | `ssl='require'` passed explicitly to create_pool; never `ssl=False`/`'disable'`. SSL is forced even though the query-string sslmode hint is stripped. |
| T-05-04 | Denial of Service | /health flooding | accept | Single-route minimal aiohttp app, no DB work per request → negligible cost; Koyeb fronts the port. Low-value target, single community. |
| T-05-05 | Tampering | asyncpg queries | mitigate | No new query code; all DB access remains `$N` parameterized (existing V5 controls in database.py). Sanitizer strips DSN query params so no unrecognized GUC reaches Postgres. |
| T-05-SC | Tampering | pip installs | mitigate | No net-new packages this plan (aiohttp/asyncpg already vetted Phases 1-4). RESEARCH Package Legitimacy Audit confirms zero new deps. |
</threat_model>
