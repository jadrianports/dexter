<objective>
Build the substrate-agnostic foundations for Player UX & Filters: the pure-logic and service-layer pieces that the Discord wiring in Plan 02 will consume. No cogs, no commands, no Discord views here — those land in Plan 02. This plan is TDD-first for everything testable.

Purpose: `/seek`, `/jump`, and `/filter` all need three primitives that do not exist yet — (1) a way to know the current playback position (no elapsed tracking exists anywhere today), (2) a time-string parser, and (3) an `AudioService` path that can emit `-ss`/`-af` while leaving the opus-passthrough default intact for ordinary playback. This plan adds exactly those, plus the config presets/caps and the personality copy the later plans reference.

Output: elapsed tracking + `active_filter` + `jump_to()` on `MusicQueue`; `parse_time()` in formatters; `get_source(..., seek_seconds=, ffmpeg_filter=)` with a pure `_build_ffmpeg_opts` helper in `AudioService`; `FFMPEG_FILTERS` + caps + cooldowns in config; new response pools; and unit tests for every pure piece.
</objective>

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Tampering / Injection | FFmpeg -af chain | mitigate | Filter chains come ONLY from the fixed `config.FFMPEG_FILTERS` dict keyed by an `app_commands.Choices` preset name (Plan 02). No user free-text ever reaches the `-af` string. `_build_ffmpeg_opts` accepts only a preset value already resolved from that dict. |
| T-07-01-02 | Tampering | -ss seek value | mitigate | `parse_time` returns an int or None; Plan 02 clamps/validates the int before it reaches `-ss`. No string interpolation of raw user text into FFmpeg args. |
| T-07-01-03 | DoS | extreme seek/filter re-encodes | accept | Re-encode cost is bounded by the existing 15-min duration cap and per-command cooldowns (SEEK/FILTER_COOLDOWN_SECONDS). Single-community scale. |
</threat_model>
