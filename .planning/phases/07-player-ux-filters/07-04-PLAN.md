## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| Discord user → user_playlists | a user writes/reads their own named snapshots |
| snapshot JSONB → queue | stored track dicts are rebuilt into Tracks and queued |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04-01 | Tampering / Injection | playlist SQL/JSONB | mitigate | $N-parameterized inserts; snapshot serialized with json.dumps; name stored as a bound param, never interpolated. |
| T-07-04-02 | Elevation of Privilege | cross-user playlists | mitigate | Every save/load/list/delete is keyed on str(interaction.user.id); no cross-user reads or deletes. |
| T-07-04-03 | DoS | oversized snapshot / unbounded playlists | mitigate | Snapshot is bounded by MAX_QUEUE_SIZE_PER_GUILD (500) on save (it mirrors the live queue) and load truncates to the same cap; PLAYLISTS_MAX_PER_USER (25) + PLAYLIST_NAME_MAX_LENGTH bound growth. |
| T-07-04-04 | Tampering | stale track URLs in a snapshot | accept | Dead URLs fail through the existing AudioService skip-and-continue path on playback. |
| T-07-04-SC | Tampering | dependencies | mitigate | No new pip deps; reuses asyncpg JSONB + Track serialization. |
</threat_model>
