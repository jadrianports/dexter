<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| Discord interaction → playback engine | Button presses and slash args from any guild member reach queue/voice control |
| User input → FFmpeg | /seek position and /filter preset reach -ss/-af option strings |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Elevation of Privilege | control buttons | mitigate | Every button callback enforces the presser is in the bot's voice channel (D-02) before any state change; otherwise ephemeral refusal, no-op. |
| T-07-02-02 | Tampering / Injection | /filter -af | mitigate | preset comes from a fixed `app_commands.Choices` set; chain is resolved from `config.FFMPEG_FILTERS` only. No free text reaches FFmpeg. |
| T-07-02-03 | Tampering | /seek -ss | mitigate | position parsed by `parse_time` to an int, range-checked against duration before reaching `-ss`. |
| T-07-02-04 | DoS | rapid button/seek/filter re-encodes | mitigate | per-command cooldowns + generation-counter cancels stale plays so spamming can't stack FFmpeg processes; existing orphan-cleanup on stop applies. |
| T-07-02-05 | Information Disclosure | ephemeral errors | mitigate | no-op/error responses are ephemeral (D-30); no internal state leaked. |
| T-07-02-SC | Tampering | dependencies | mitigate | No new pip deps; discord.py 2.3 already provides persistent views / Choices. |
</threat_model>
