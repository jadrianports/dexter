<objective>
Make the invisible fire-and-forget background tasks surface their exceptions (REL-02 / D-03, D-04).

This plan creates a shared `utils/tasks.py` `make_task` helper whose done-callback (a) always logs
the exception to `dexter.log` and (b) posts it to the Discord error channel, rate-limited so a
recurring per-track crash cannot flood the channel. It then replaces the three genuinely-bare
`asyncio.create_task` sites in `cogs/music.py` (`_post_auto_lyrics`, `_prefetch_next_track`,
`try_auto_queue`) with `make_task`, while deliberately leaving the `_play_track` tasks untouched
(they already have full internal handling — adding callbacks would double-log handled errors).

Purpose: failures in prefetch / auto-lyrics / auto-queue currently log at debug or are swallowed
entirely, so the user (running on the PC) can't see them without tailing logs. D-03 wants real-time
Discord visibility; D-04 wants it throttled so it never spams.
Output: `utils/tasks.py` + three wired call sites + unit tests for the callback.
</objective>
