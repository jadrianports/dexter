## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external monitor ‚Üí `/health` HTTP | Unauthenticated external reader (UptimeRobot / future Pi) reads the degraded body |
| Discord user ‚Üí `/leaderboard`, `/stats` | Slash-command input triggers DB-heavy queries that may time out |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | `/health` degraded body (bot.py health handler) | mitigate | Body limited to generic reason strings ("MusicCog not loaded", "database unreachable", "discord gateway not ready") + `status` key ‚Äî no guild/shard/pool internals, no secrets, no stack traces (D-27 preserved; existing test_health_endpoint assertions enforce key-set) |
| T-09-02 | Information Disclosure | DB timeout message in `/leaderboard`, `/stats` | mitigate | TimeoutError branch emits a static lowercase string; never interpolates the exception, SQL, or DSN; log records command name only |
| T-09-03 | Denial of Service | Slow DB query blocking the event loop | mitigate | Pool-wide `command_timeout=config.DB_COMMAND_TIMEOUT_SECONDS` caps every query as a floor; handler converts the timeout into a fast user-facing response |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase (RESEARCH Package Legitimacy Audit: none) |
</threat_model>
