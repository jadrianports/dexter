<objective>
Make startup un-wedgeable and resilient, and surface background-loop failures (REL-04, REL-03, REL-02).

Three bot.py edits: (1) wrap `_initialize_once()` in `asyncio.wait_for` so a true hang becomes a
`TimeoutError` that runs the `finally` and resets the `_ready_initializing` guard â€” closing the
REL-04 hole where a non-raising hang permanently blocks all future ready handling (D-06);
(2) wrap `bot.tree.sync()` in a timeout with a single-flight background retry so a slow/failed sync
never blocks the bot from coming online with its already-registered commands (REL-03 / D-05);
(3) add `@loop.error` handlers to the four `@tasks.loop` loops so their exceptions log and post a
throttled embed instead of dying silently (REL-02 loop path / D-03, D-04).

Purpose: today `_ready_initializing` only resets in `finally` on a RAISED exception â€” a coroutine
that hangs (e.g. a stuck cold-Postgres connect) never reaches `finally` and dead-locks ready
handling forever. And `bot.tree.sync()` is currently bare in `first_run`. This plan removes both
hang surfaces. All edits are in bot.py, so this plan owns bot.py for Wave 2.
Output: a startup path that always finishes, recovers its command sync, and reports loop crashes.
</objective>
