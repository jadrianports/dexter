<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| song_history rows → aggregate result | Bot-owned table; the risk is a helper accidentally spanning users/guilds, leaking one user's taste into another's episode. |
| memory_id → expires_at UPDATE | A wrong/unbounded UPDATE could alter rows beyond the intended memory or clobber other columns. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-02 | Information disclosure | get_user_artist_activity | mitigate | `WHERE guild_id = $1 AND user_id = $2` positional scope; test asserts cross-guild isolation (Pitfall 4/6). |
| T-13-03 | Denial of service | get_active_taste_users / get_user_artist_activity | mitigate | `queued_at > $N` window + baseline bound keep scans index-aligned (idx_history_guild/idx_history_user); no cross-join; runs once daily off-peak (plan 13-04 schedule). |
| T-13-04 | Tampering | refresh_memory_expiry | mitigate | UPDATE is `WHERE id = $1`, sets only `expires_at = $2`; docstring + test assert hit_count/salience untouched, so Phase 11 rows can never be side-effected beyond their horizon. |
| T-13-05 | Tampering (SQL injection) | all three helpers | mitigate | Every input bound as `$N` positional param; no string interpolation (T-12-02-03). |
