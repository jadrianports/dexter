<objective>
Deliver MEM-06: a surfaced memory gains durability by having its `expires_at` pushed out on the
recall path, so a fact that keeps proving relevant outlives an equally-old never-surfaced one under
the daily decay sweep. Reinforcement is **expiry-only** (D-01) and happens at the **single
`recall()` -> `bump_surfaced` chokepoint** (D-02), grouped by each fact's own kind via the existing
pure `resolve_decay_days` resolver.

Purpose: This is the "more durable" half of "smarter memory." Picking expiry-extension over a
salience bump keeps *importance* (salience) orthogonal to *durability* (recency of usefulness) and,
critically, means `salience` — the field every other subsystem keys on — is never touched by recall,
making the SC-3 byte-identical guarantee structurally easy to prove. This plan carries the entire
SC-3 regression risk of the phase (the subsystem holds the Phase 13 CR-01 and Phase 21 guild-scoping
scars), so the regression guard and the one-test monkeypatch fix are explicit tasks.

Output: one new `database.py` helper (`reinforce_memory_expiry`), a ~10-line `recall()` step-7b
addition, an extended `TestRecallService`, and a new live-DB test file `tests/test_database_phase25.py`
(SC-1 sweep-ordering + SC-3 non-mutation). No new table, no schema change, no `MemoryFact` field.
</objective>
