<objective>
Upgrade the `services/youtube.py` download pipeline: add SponsorBlock segment removal (PERF-07) and codec-path logging (PERF-02 / D-03) to the existing `DOWNLOAD_OPTS` and `download()`. Per 06-RESEARCH.md's critical finding, copy-when-opus (D-01) is ALREADY performed by `FFmpegExtractAudioPP` (`preferredcodec='opus'` copies when source is opus) — so this plan does NOT rewrite download logic. Keeping `preferredcodec='opus'` also guarantees every cache file stays opus, so the existing `FFmpegOpusAudio` passthrough in `get_source` always holds (D-02); we do not store raw non-opus bestaudio. The real work is: (1) add the two SponsorBlock postprocessors in the correct order, and (2) add a `postprocessor_hooks` closure that logs whether the codec path was copy or transcode, plus timing.

Purpose: Make PERF-02 verifiable (logs prove copy vs transcode) and deliver PERF-07 (segments baked out at download time, playback stays pure passthrough). Segments are removed with no user notification — silent skip (D-15). Respects D-16: tracks with cut segments re-encode even on opus sources — that is expected.
Output: modified `services/youtube.py` `DOWNLOAD_OPTS` + `download()`; new test classes in `tests/test_youtube.py`.
</objective>
