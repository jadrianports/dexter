<objective>
Optimize the `services/audio.py` playback/cache layer with two independent changes: (1) wire the existing-but-unused `DOWNLOAD_TIMEOUT_SECONDS` into `get_source` tier 2 via `asyncio.wait_for`, falling back to the existing stream tier on timeout (PERF-04 / D-10 / D-11); and (2) rewrite `cleanup_cache()` from `st_atime` sort to play-frequency LFU sourced from `song_history`, protecting in-use tracks (PERF-05 / D-12 / D-13).

Purpose: A slow download must never hang a waiting user (on-demand keeps the 10s budget → stream), and the cache must stop evicting recently/about-to-play songs — including the just-prefetched track. These are the two `services/audio.py`-owned PERF requirements; they share no files with Plan 02 (youtube.py) and run in the same wave.
Output: modified `services/audio.py` `get_source` + `cleanup_cache`; updated `tests/test_audio.py`.
</objective>

| T-06-05 | Denial of Service | Hung download blocking a waiting user | mitigate | `asyncio.wait_for(..., DOWNLOAD_TIMEOUT_SECONDS=10)` bounds the on-demand download; on timeout it falls back to the stream tier so playback continues (D-10/D-11) |
| T-06-06 | Tampering | LFU play_count query (song_history.url) | mitigate | `SELECT ... GROUP BY url` uses no user-supplied params; video_id parsed locally from stored URLs; no interpolation. Eviction never deletes a protected (playing/queued/prefetched) file (D-12) — prevents the self-defeating evict-the-prefetch bug (Pitfall 6) |
