## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| song_history titles/artists (user-influenced YouTube metadata) → Gemini prompt text | recommendation/jam/discover prompts embed real track metadata |
| Gemini reply → parse_suggestions → downstream validation | model-generated {title, artist} strings are untrusted until validated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-04 | Tampering | `build_recommendation_prompt` / `build_jam_suggestion_prompt` / `build_discover_commentary_prompt` | accept | Prompt-injection via song titles/artist names is an EXISTING surface unchanged by this phase — no new trust boundary crossed; the recent-songs cleaning (`build_genius_search_query`, cogs/ai.py) already runs upstream. Low value target, sarcastic-bot context |
| T-14-03 | Spoofing (content) | jam/recommendation suggestion output | mitigate | Prompt builders only FORMAT text; every model-suggested track is re-validated against real YouTube search via `validate_youtube_match` at the cog layer (plans 14-03/14-05) before being offered or queued — builders never trusted to produce real tracks |
| T-14-02 | Information Disclosure | `select_positive_taste_context` | mitigate | Returns an UNATTRIBUTED flat list[str]; carries no member identity, so the caller cannot leak "member X likes Y" into a shared-queue prompt (Pitfall 4). Enforced by the round-robin/dedup unit tests |
| T-14-SC | Tampering | package installs | accept | Zero new package installs this phase (RESEARCH Package Legitimacy Audit = N/A) |
