<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Discord interaction → /memory subcommands | invoker identity (`str(interaction.user.id)`) is the ONLY scope; no target/id args accepted |
| Confirm view → destructive DELETE | a button press crosses into an irreversible nuke-all; double-press / cross-user are the risks |
| Fact text → Discord embed | verbatim stored facts (bot-distilled banter) render to the channel; mention-injection is the defense-in-depth concern |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-08 | Information Disclosure | `/memory view` viewing another user's facts | mitigate | No `target` param on `memory_view`; always `str(interaction.user.id)`; response `ephemeral=True` so even own view isn't visible to others. Signature test enforces no target param. |
| T-15-09 | Tampering | `/memory forget` deleting another user's facts | mitigate | No `target` param on `memory_forget`; delete goes through single-param `delete_all_user_memories(pool, user_id)`. Signature tests on both the command and the helper (15-01) prevent a cross-user variant. |
| T-15-10 | Tampering (correctness) | confirm double-press → double DELETE | mitigate | `_used` guard + immediate `child.disabled = True` before async work, cloned from JamSuggestConfirmView (proven). Second DELETE is a harmless no-op but guarded regardless. |
| T-15-11 | Information Disclosure | mention injection via fact text in the embed | mitigate | `AllowedMentions.none()` on MemoryPageView edits (cloned from LyricsPageView), defense-in-depth. |
| T-15-12 | Information Disclosure | view truncated below what forget erases | mitigate | `list_user_memories` called with `config.MEMORY_MAX_PER_USER`; test asserts the cap (Pitfall 2). |
| T-15-SC | Tampering (supply chain) | pip installs | accept | Zero new pip dependencies this phase; new cog composes only already-installed discord.py + asyncpg patterns. |
</threat_model>
