<objective>
Clear the three things in Phase 23 that a Claude agent genuinely **cannot** do, and clear them **early**, so no downstream plan stalls waiting on a human mid-execution.

This phase has exactly three human-only dependencies, and each blocks a different downstream plan:

1. **The verbatim Dexter transcript (D-06).** RESEARCH Finding 1 settled this with code-level proof: `services/gemini.py::chat()` logs `len(response.text or '')` — the character *count* — and never the text, at every call site. So `logs/dexter.log` cannot supply the lines, and D-06 forbids an executor writing them. The user is the only source. Blocks plan 23-06 (the demo mock) and 23-07 (the README GIF).
2. **The package-legitimacy gate.** `astro` (npm) and `playwright` (PyPI) are the first two third-party toolchains to enter this Python repo. RESEARCH's Package Legitimacy Audit ran no slopcheck, so both are `[ASSUMED]` — which requires a blocking human verification before the first install. Blocks plan 23-03 (`npm install`) and 23-07 (`pip install playwright`).
3. **GitHub Pages enablement.** RESEARCH Finding 4 confirmed, by reading `actions/configure-pages`'s own `action.yml`, that `enablement: true` requires a PAT or GitHub App token — not the default `GITHUB_TOKEN`. Adding a PAT purely to automate a one-time toggle would break the zero-secrets-in-CI posture that Phase 22 D-04 exists to preserve. Blocks plan 23-04's `pages.yml`, which fails on a missing `github-pages` environment until it is done.

Output: a committed `23-DEMO-TRANSCRIPT.md` holding the real lines, two verified packages, and a Pages-enabled repo.

**This plan is entirely checkpoints. It writes no product code.**
</objective>
