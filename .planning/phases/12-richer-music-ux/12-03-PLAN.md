<objective>
Implement UX-03: add LRCLIB (`lrclib.net`) as the third lyrics fallback so `/lyrics` degrades
gracefully when both Genius and AZLyrics miss. The chain becomes Genius → AZLyrics → LRCLIB.
LRCLIB is free, keyless, and purpose-built for music players; we use its `/api/search`
endpoint and the `plainLyrics` field, stripping any embedded LRC metadata headers before
display (D-10, D-11).

Purpose: a last-resort lyrics source that is robust to scrape fragility (D-10).
Output: a `strip_lrc_headers` pure helper, the `_get_lrclib` fetch method, the chain wiring in
`get_lyrics`, and unit tests (pure header-stripping + mocked fetch).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| lrclib.net HTTP response → bot | Untrusted external JSON/lyrics crosses into a Discord embed |
| (title, artist) → LRCLIB request | Search params cross into an outbound HTTP request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03-01 | Spoofing | LRCLIB request URL | mitigate | Host hard-coded as `_LRCLIB_BASE = "https://lrclib.net"`; title/artist passed via aiohttp `params=` (URL-encoded) — no user-supplied host, no SSRF |
| T-12-03-02 | DoS | oversized / slow LRCLIB response | mitigate | aiohttp.ClientTimeout(total=10) + 500_000-byte response cap (mirrors AZLyrics) |
| T-12-03-03 | Tampering | XSS / @mention injection in returned lyrics | mitigate | sanitize_lyrics() strips HTML and neutralizes @everyone/@here before the text is displayed |
| T-12-03-04 | Information Disclosure | LRC metadata headers leaking into output | mitigate | strip_lrc_headers() removes `[ti:]/[ar:]/...` lines before sanitize/display (Pitfall 1) |
| T-12-03-05 | DoS | LRCLIB throttling / no documented UA contract (A1) | mitigate | Send a descriptive User-Agent; treat non-200 (incl. 429) as None and fall through to "no lyrics" — bounded, no retry storm |
| T-12-03-SC | Tampering | package installs | n/a | No new packages — aiohttp already installed; LRCLIB accessed via raw HTTP, no SDK |

No HIGH-severity threats remain unmitigated.
