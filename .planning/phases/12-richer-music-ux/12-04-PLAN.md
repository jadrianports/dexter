<objective>
Implement UX-04: stop AI auto-queue from queueing hallucinated tracks. Today
`try_auto_queue` blindly trusts the single top YouTube hit for each Gemini suggestion. This
plan widens the search to several candidates, validates each against the suggestion with a
pure fuzzy title+artist matcher, takes the first that passes, and — when none pass — falls
through to the next Gemini suggestion (still capped at `AUTO_QUEUE_SONGS_PER_ROUND` tracks
added), per D-12, D-13, D-14.

Purpose: makes auto-queue trustworthy by rejecting clear mismatches before they reach the
queue.
Output: a pure `logic/autoqueue.py` validator, the `AUTO_QUEUE_SEARCH_CANDIDATES` knob, the
`try_auto_queue` loop mutation, and unit tests (pure validator + mocked fall-through loop).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Gemini suggestion text → auto-queue | Untrusted LLM output drives a YouTube search |
| YouTube search result → queue | A result title/URL crosses into the live playback queue |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-04-01 | Spoofing / Tampering | hallucinated suggestion queued as a wrong/arbitrary track | mitigate | validate_youtube_match requires BOTH title+artist token sets to appear in the actual YouTube result title before the result is queued; mismatches are rejected and the suggestion is skipped |
| T-12-04-02 | Tampering | Gemini-supplied URL injected into the queue | mitigate | Suggestions are used ONLY as async_search query strings; the queued Track is built from async_extract of a YouTube search-result URL — never from a raw Gemini-provided URL |
| T-12-04-03 | DoS | widened search amplifies work | mitigate | AUTO_QUEUE_SEARCH_CANDIDATES bounded (=3) and AUTO_QUEUE_SONGS_PER_ROUND / AUTO_QUEUE_MAX_ROUNDS caps preserved; loop breaks once the round is full |
| T-12-04-04 | Tampering | validator non-determinism / injection | mitigate | validate_youtube_match is pure (stdlib re only), deterministic, no I/O — no injection surface; unit-tested |
| T-12-04-SC | Tampering | package installs | n/a | No new packages — stdlib re only; reuses existing youtube_service |

No HIGH-severity threats remain unmitigated.
