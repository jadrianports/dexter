<objective>
Implement UX-01: a guild-scoped shared-playlist system ("jams") exposed as a new `/jam`
command group, distinct from the existing user-global `/playlist` group. Jams are a
collaborative server mixtape — anyone in the guild can edit, keyed on `guild_id`, with no
ownership gate.

Purpose: gives each server a shared, growing collection of tracks separate from any one
user's favorites (D-01..D-05).
Output: `guild_jams` table + 5 DB helpers, the `/jam` group (save/add/load/list/delete),
the `JAMS_PER_GUILD_MAX` config knob, and unit/integration tests.

Note on shared files: this plan APPENDS a new table block + 5 new helpers to `database.py`
and APPENDS one knob to `config.py` (in a new `# --- Phase 12 ---` section). It edits no
existing lines in those files, so it does not collide with the other Wave-1 plans that also
append to them.
</objective>

| /jam load → MusicQueue | Stored snapshot crosses into the live playback queue |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01-01 | Tampering | jam name → guild_jams SQL | mitigate | All guild_jams helpers bind values as $1/$2/$3 positional params (mirror save_playlist); no f-string SQL |
| T-12-01-02 | Information Disclosure | cross-guild jam reads | mitigate | Every read/write keyed on `guild_id = $1`; integration test asserts guild-A jam invisible to guild-B |
| T-12-01-03 | Tampering / DoS | oversized / empty jam name | mitigate | Reject empty after strip(); enforce PLAYLIST_NAME_MAX_LENGTH (60) before any write (reuse playlist guard) |
| T-12-01-04 | Elevation of Privilege | no creator/role gate on edits (D-03) | accept | Intentional collaborative "server mixtape" model; scope limited to in-guild members — Discord already gates the command to guild members; no PII, low-value target |
| T-12-01-05 | DoS | /jam load onto a near-full queue | mitigate | queue.add wrapped in try/except QueueFullError honoring MAX_QUEUE_SIZE_PER_GUILD; truncation reported, never silent (Pitfall 7) |
| T-12-01-SC | Tampering | package installs | n/a | This plan installs no new packages (stdlib + already-installed asyncpg/discord.py only) |

No HIGH-severity threats remain unmitigated.
