## Overview

The complete first release of Dexter ("Dex") ‚Äî a sarcastic, personality-driven Discord bot. v1.0 takes the bot from a reliable YouTube music player, through a Gemini-powered personality with `/ask` and `/imagine`, a production-honest hardening pass, an "alive" layer of unprompted roasts/reactions/lyrics/history, and finally a scale pass (PostgreSQL, sharding, queue persistence, and a resolved Oracle Cloud A1 hosting decision). All 45 v1 requirements are satisfied at the code/structural level; live-deploy UAT is carried forward as the day-1 deployment checklist (see Deferred Items in STATE.md).

## Phases

### Phase 1: Music MVP

**Goal**: A working sarcastic-branded YouTube music player with a robust per-server queue
**Depends on**: Nothing (first phase)
**Requirements**: MUSIC-01 ‚Ä¶ MUSIC-11
**Plans**: Shipped pre-GSD

Success Criteria (all TRUE):

1. `/play` by search (5-result select menu) or direct URL plays in voice
2. Playlist URLs queue up to 50 tracks, truncated + informed beyond 50
3. `/skip`, `/pause`, `/resume`, `/stop`, `/queue`, `/shuffle`, `/loop`, `/nowplaying`, `/replay`, `/help` all functional
4. Songs > 15 min and livestreams rejected with personality; audio plays cache-first with stream fallback
5. Auto-leaves voice after 10 min idle and clears the queue

**Details:** Per-server queue uses `current_index` (no popping) with generation-counter race prevention; cache-first audio (`{video_id}.opus`) with 2GB atime eviction.

### Phase 2: Personality + AI

**Goal**: Dexter gains its Gemini-powered personality, conversational `/ask`, image generation, and self-feeding music recommendations
**Depends on**: Phase 1
**Requirements**: AI-01 ‚Ä¶ AI-07, IMG-01 ‚Ä¶ IMG-03, PERS-01
**Plans**: Shipped pre-GSD

Success Criteria (all TRUE):

1. `/ask <question>` returns in-character Gemini response using last 10 messages + taste summary
2. Mood shifts with daily command count (15/30/50 thresholds); seasonal context injected
3. `/imagine <prompt>` returns a `gemini-2.5-flash-image` image (or personality refusal), capped 10/day

**Details:** `gemini-2.0-flash` for chat; global sliding-window rate limiter (priority 1 = user, priority 2 = background).

## Milestone Summary

**Decimal Phases:**

- Phase 2.5: Hardening ‚Äî inserted between Phase 2 and Phase 3 as a production-honest reliability pass before adding "alive" surface area.

**Key Decisions:**

- Image-gen model = `gemini-2.5-flash-image` (resolved during ingest via ground-truth tie-break; matches shipped `config.py:36`)
- Layered cog ‚Üí service ‚Üí model architecture; services wired in `bot.py`, accessed via `self.bot`
- `current_index` queue (no popping) ‚Äî enables `/replay`, loop wrap, `/history`
- Global Gemini rate limiter with priority tiers (sliding-window deque) ‚Äî one shared 15 RPM budget
- Phase 2.5 parks live-concurrency bugs rather than fixing blind (local-boot-only verification discipline)
- Hosting resolved (Phase 4) ‚Üí Oracle Cloud Always Free A1 ARM; Docker Compose for Hetzner portability
- Persistence migrated SQLite ‚Üí PostgreSQL (asyncpg 0.31.0; raw `CREATE TABLE IF NOT EXISTS` over Alembic)
- `MAX_QUEUE_SIZE_PER_GUILD=500`; cap enforced in `MusicQueue.add()` so the playlist loop is covered at source

**Issues Resolved:**

- DB write contention under concurrency ‚Üí `log_track_batch` single-transaction batching
- Unbounded queues ‚Üí `QueueFullError` cap at 500
- Message-buffer growth ‚Üí `_evict_stale` TTL (24h) eviction
- 24/7 hosting question (open since Phase 2.5) ‚Üí Oracle A1 + keepalive/dead-man + pg_dump backup

**Issues Deferred (carried to live deploy / next milestone):**

- Live-deploy UAT for Phases 3 & 4 (9 + 6 behavioral checks) ‚Äî require Oracle VM + Postgres + Discord gateway
- Live-concurrency reconnect race (`cogs/music.py:~609`) ‚Äî parked for a dedicated live `/gsd:debug` session
- Idle-cleared / reconnect-failure queues don't call `clear_persisted()` (IN-02) ‚Äî restored on next boot; behavioral, not a crash
- WR-02/03/04/05/07 minor follow-ups documented in `04-REVIEW.md`

**Technical Debt Incurred:**

- A handful of unused imports (IN-01) flagged, non-functional
- `backup.sh` pipe masks `pg_dump` failure (WR-07) ‚Äî to harden during live backup setup
